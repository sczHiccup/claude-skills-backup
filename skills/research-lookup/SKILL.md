# 研究信息查找

## 概述

此技能通过 OpenRouter 使用 Perplexity 的 Sonar 模型实现实时研究信息查找。它根据查询复杂性智能选择 **Sonar Pro Search**（快速、高效查找）和 **Sonar Reasoning Pro**（深度分析推理）。该技能提供对当前学术文献、最新研究、技术文档和带适当引用和来源归属的一般研究信息的访问。

## 何时使用此技能

当您需要时，请使用此技能：

- **当前研究信息**：特定领域的最新研究、论文和发现
- **文献验证**：检查事实、统计数据或声明与当前研究的一致性
- **背景研究**：为科学写作收集上下文和支持证据
- **引用来源**：查找相关论文和研究以在手稿中引用
- **技术文档**：查找规格、协议或方法
- **最新动态**：跟进新兴趋势和突破
- **统计数据**：查找最新统计数据、调查结果或研究发现
- **专家意见**：获取最近访谈、评论或评论的见解

## 核心能力

### 1. 学术研究查询

**搜索学术文献**：查询特定领域的最新论文、研究和评论：

```
查询示例：
- "2024年CRISPR基因编辑的最新进展"
- "阿尔茨海默病治疗的最新临床试验"
- "药物发现中的机器学习应用系统评价"
- "气候变化对生物多样性的影响元分析"
```

**预期响应格式**：
- 最近文献的关键发现摘要
- 3-5篇最相关论文的引用，包括作者、标题、期刊和年份
- 突出显示的关键统计数据或发现
- 研究空白或争议的识别
- 可用时的完整论文链接

### 2. 技术和方法信息

**协议和方法查找**：查找详细的程序、规格和方法：

```
查询示例：
- "蛋白质检测的西方印迹协议"
- "RNA测序文库准备方法"
- "临床试验的统计功效分析"
- "机器学习模型评估指标"
```

**预期响应格式**：
- 步骤详细的程序或协议
- 所需材料和设备
- 关键参数和注意事项
- 常见问题的故障排除
- 对标准协议或经典论文的参考

### 3. 统计和数据信息

**研究统计**：查找当前统计数据、调查结果和研究数据：

```
查询示例：
- "2024年美国人口糖尿病的流行率"
- "全球可再生能源采用统计"
- "各国COVID-19疫苗接种率"
- "医疗行业的AI采用调查"
```

**预期响应格式**：
- 带日期和来源的当前统计数据
- 数据收集的方法论
- 可用时的置信区间或误差范围
- 与前几年或基准的比较
- 原始调查或研究的引用

### 4. 引用和参考帮助

**引用查找**：查找相关论文和研究以在手稿中引用：

```
查询示例：
- "变压器架构的基础论文"
- "量子计算的经典作品"
- "气候变化缓解的关键研究"
- "癌症免疫疗法的里程碑试验"
```

**预期响应格式**：
- 5-10篇最有影响力或相关的论文
- 完整的引用信息（作者、标题、期刊、年份、DOI）
- 每篇论文贡献的简要描述
- 可用时的引用影响指标（h-index，引用计数）
- 期刊影响因子和排名

## 自动模型选择

此技能具有基于查询复杂性的 **智能模型选择**：

### 模型类型

**1. Sonar Pro Search** (`perplexity/sonar-pro-search`)
- **用例**：简单信息查找
- **最佳适用**：
  - 简单的事实查找查询
  - 最近的出版物搜索
  - 基本协议查找
  - 统计数据检索
- **速度**：快速响应
- **成本**：每次查询成本较低

**2. Sonar Reasoning Pro** (`perplexity/sonar-reasoning-pro`)
- **用例**：需要深度推理的复杂分析查询
- **最佳适用**：
  - 比较分析（"比较X与Y"）
  - 多项研究的综合
  - 评估权衡或争议
  - 解释机制或关系
  - 批判性分析和解释
- **速度**：较慢但更全面
- **成本**：每次查询成本较高，但提供更深的见解

### 复杂性评估

该技能使用以下指标自动检测查询复杂性：

**推理关键词**（触发 Sonar Reasoning Pro）：
- 分析性：`compare`, `contrast`, `analyze`, `analysis`, `evaluate`, `critique`
- 比较性：`versus`, `vs`, `vs.`, `compared to`, `differences between`, `similarities`
- 综合性：`meta-analysis`, `systematic review`, `synthesis`, `integrate`
- 因果性：`mechanism`, `why`, `how does`, `how do`, `explain`, `relationship`, `causal relationship`, `underlying mechanism`
- 理论性：`theoretical framework`, `implications`, `interpret`, `reasoning`
- 辩论性：`controversy`, `conflicting`, `paradox`, `debate`, `reconcile`
- 权衡性：`pros and cons`, `advantages and disadvantages`, `trade-off`, `tradeoff`, `trade offs`
- 复杂性：`multifaceted`, `complex interaction`, `critical analysis`

**复杂性评分**：
- 推理关键词：每个3分（权重较大）
- 多个问题：每个问号2分
- 复杂句子结构：每个从句指示符（and, or, but, however, whereas, although）1.5分
- 非常长的查询：如果超过150个字符则1分
- **阈值**：得分≥3分的查询触发 Sonar Reasoning Pro

**实际结果**：即使是一个强推理关键词（compare, explain, analyze 等）也会触发更强大的 Sonar Reasoning Pro 模型，确保在需要时获得深度分析。

**示例查询分类**：

✅ **Sonar Pro Search**（简单查找）：
- "2024年CRISPR基因编辑的最新进展"
- "美国人口糖尿病的流行率"
- "蛋白质检测的西方印迹协议"

✅ **Sonar Reasoning Pro**（复杂分析）：
- "比较和对比mRNA疫苗与传统疫苗在癌症治疗中的优势和局限性"
- "解释肠道微生物群与抑郁症之间关系的机制"
- "分析围绕AI在医学诊断中的争议并评估权衡"

### 手动覆盖

您可以使用 `force_model` 参数强制使用特定模型：

```python
# 强制使用 Sonar Pro Search 进行快速查找
research = ResearchLookup(force_model='pro')

# 强制使用 Sonar Reasoning Pro 进行深度分析
research = ResearchLookup(force_model='reasoning')

# 自动选择（默认）
research = ResearchLookup()
```

命令行用法：
```bash
# 强制使用 Sonar Pro Search
python research_lookup.py "your query" --force-model pro

# 强制使用 Sonar Reasoning Pro
python research_lookup.py "your query" --force-model reasoning

# 自动（无标志）
python research_lookup.py "your query"
```

## 技术集成

### OpenRouter API 配置

此技能与 OpenRouter (openrouter.ai) 集成，以访问 Perplexity 的 Sonar 模型：

**模型规格**：
- **模型**：
  - `perplexity/sonar-pro-search`（快速查找）
  - `perplexity/sonar-reasoning-pro-online`（深度分析）
- **搜索模式**：学术/学术模式（优先考虑同行评审的来源）
- **搜索上下文**：始终使用 `high` 搜索上下文以获得更深、更全面的研究结果
- **上下文窗口**：200K+ tokens 以支持全面研究
- **功能**：学术论文搜索、引用生成、学术分析
- **输出**：带有引用和来源链接的丰富响应，来自学术数据库

**API 要求**：
- OpenRouter API 密钥（设置为 `OPENROUTER_API_KEY` 环境变量）
- 具有足够积分的账户以进行研究查询
- 正确归属和引用来源

**学术模式配置**：
- 系统消息配置为优先考虑学术来源
- 搜索集中在同行评审的期刊和学术出版物上
- 增强的引用提取以支持学术参考
- 优先考虑最近的学术文献（2020-2024）
- 直接访问学术数据库和存储库

### 响应质量和可靠性

**来源验证**：该技能优先考虑：
- 同行评审的学术论文和期刊
- 信誉良好的机构来源（大学、政府机构、非政府组织）
- 最近的出版物（优先考虑过去2-3年）
- 高影响力的期刊和会议
- 原始研究优于二手来源

**引用标准**：所有响应均包含：
- 完整的书目信息
- DOI 或稳定的 URL（如有）
- 网络来源的访问日期
- 清晰的直接引用或数据归属

## 查询最佳实践

### 1. 模型选择策略

**对于简单查找（Sonar Pro Search）**：
- 特定主题的最新论文
- 统计数据或流行率
- 标准协议或方法
- 针对特定论文的引用查找
- 事实信息检索

**对于复杂分析（Sonar Reasoning Pro）**：
- 比较研究和综合
- 机制解释
- 争议评估
- 权衡分析
- 理论框架
- 多方面关系

**专业提示**：自动选择已针对大多数用例进行了优化。只有在您有特定要求或知道查询需要比检测到的更深入的推理时，才使用 `force_model`。

### 2. 具体且集中的查询

**良好的查询**（将触发适当模型）：
- "2023-2024年癌症治疗的mRNA疫苗随机对照试验" → Sonar Pro Search
- "比较mRNA疫苗与传统疫苗在癌症治疗中的有效性和安全性" → Sonar Reasoning Pro
- "解释CRISPR脱靶效应发生的机制及减少策略" → Sonar Reasoning Pro

**差的查询**：
- "告诉我关于AI的事"（过于宽泛）
- "癌症研究"（缺乏具体性）
- "最新消息"（过于模糊）

### 3. 结构化查询格式

**推荐结构**：
```
[主题] + [具体方面] + [时间范围] + [信息类型]
```

**示例**：
- "CRISPR基因编辑 + 脱靶效应 + 2024 + 临床试验"
- "量子计算 + 错误校正 + 最新进展 + 综述论文"
- "可再生能源 + 太阳能效率 + 2023-2024 + 统计数据"

### 4. 后续查询

**有效的后续查询**：
- "给我查看Smith et al. 2024论文的完整引用"
- "该方法的局限性是什么？"
- "查找使用不同方法的类似研究"
- "在该研究领域存在哪些争议？"

## 错误处理和限制

**已知限制**：
- 信息截止：响应受限于训练数据（通常为2023-2024）
- 付费内容：可能无法访问付费墙后的全文
- 新兴研究：可能错过尚未索引的非常新论文
- 专业数据库：无法访问专有或受限数据库

**错误条件**：
- API 速率限制或配额超出
- 网络连接问题
- 格式错误或模糊的查询
- 模型不可用或维护中

**回退策略**：
- 重新措辞查询以提高清晰度
- 将复杂查询拆分为更简单的组件
- 如果最近的数据不可用，请使用更广泛的时间范围
- 交叉引用多个查询变体

## 性能和成本考虑

### 响应时间

**Sonar Pro Search**：
- 典型响应时间：5-15秒
- 最适合快速信息收集
- 适合批量查询

**Sonar Reasoning Pro**：
- 典型响应时间：15-45秒
- 对于复杂分析查询值得等待
- 提供更全面的推理和综合

### 成本优化

**自动选择的好处**：
- 通过对简单查询使用 Sonar Pro Search 节省成本
- 为真正需要深入分析的查询保留 Sonar Reasoning Pro
- 优化成本与质量之间的平衡

**手动覆盖用例**：
- 当预算有限且速度为优先时，强制使用 Sonar Pro Search
- 当处理需要最大深度的关键研究时，强制使用 Sonar Reasoning Pro
- 用于论文的特定部分（例如，方法使用 Pro Search，讨论使用 Reasoning）

**最佳实践**：
1. 对于大多数用例信任自动选择
2. 审查查询结果 - 如果 Sonar Pro Search 提供的深度不足，请使用推理关键词重新措辞
3. 战略性地使用批量查询 - 组合简单查找以最小化总查询数量
4. 对于文献综述，首先使用 Sonar Pro Search 获取广度，然后使用 Sonar Reasoning Pro 进行综合

## 安全和伦理考虑

**负责任的使用**：
- 尽可能验证所有信息与原始来源的一致性
- 清晰地归属所有数据和引用给原始来源
- 避免将 AI 生成的摘要呈现为原创研究
- 尊重版权和许可限制
- 用于研究辅助，而不是绕过付费墙或订阅

**学术诚信**：
- 始终引用原始来源，而不是 AI 工具
- 作为文献搜索的起点使用
- 遵循机构关于 AI 工具使用的指导
- 保持研究方法的透明度
