# 科学写作

## 概述

**这是深度研究和写作工具的核心技能**——将 AI 驱动的深度研究与格式良好的书面输出相结合。每个生成的文档都经过全面的文献搜索和通过 research-lookup 技能验证的引用支持。

科学写作是以精确和清晰的方式传达研究的过程。使用 IMRAD 结构、引用（APA/AMA/Vancouver）、图表和报告指南（CONSORT/STROBE/PRISMA）撰写手稿。将此技能应用于研究论文和期刊提交。

**关键原则：始终以完整段落和流畅的散文进行写作。绝不要在最终手稿中提交项目符号。** 使用两阶段过程：首先利用 research-lookup 创建包含关键点的章节大纲，然后将这些大纲转换为完整段落。

## 何时使用此技能

当您：
- 写作或修订科学手稿的任何部分（摘要、引言、方法、结果、讨论）
- 使用 IMRAD 或其他标准格式构建研究论文
- 以特定样式格式化引用和参考文献（APA、AMA、Vancouver、Chicago、IEEE）
- 创建、格式化或改进图表和数据可视化
- 应用特定研究的报告指南（CONSORT 用于试验，STROBE 用于观察性研究，PRISMA 用于综述）
- 起草符合期刊要求的摘要（结构化或非结构化）
- 准备提交给特定期刊的手稿
- 提高写作的清晰度、简洁性和准确性
- 确保正确使用领域特定的术语和命名法
- 处理审稿人评论并修订手稿

## 使用科学示意图进行视觉增强

**⚠️ 强制性：每篇科学论文必须包含一个图形摘要以及 1-2 个使用 scientific-schematics 技能生成的附加 AI 图形。**

这不是可选的。没有视觉元素的科学论文是不完整的。在最终确定任何文档之前：
1. **始终生成一个图形摘要** 作为第一个视觉元素
2. 至少生成一个使用 scientific-schematics 的附加示意图或图表
3. 对于全面的论文，优选 3-4 个总图形（图形摘要 + 方法流程图 + 结果可视化 + 概念图）

**每篇科学写作必须包含一个图形摘要。** 这是您论文的视觉摘要：
- 出现在文本摘要之前或紧接其后
- 用一张图像捕捉整篇论文的关键消息
- 适合期刊目录展示
- 使用横向方向（通常为 1200x600px）

**首先生成图形摘要：**
```bash
python scripts/generate_schematic.py "Graphical abstract for [paper title]: [brief description showing workflow from input → methods → key findings → conclusions]" -o figures/graphical_abstract.png
```

**图形摘要要求：**
- **内容**：展示工作流程、关键方法、主要发现和结论的视觉摘要
- **风格**：干净、专业，适合期刊目录
- **元素**：包括 3-5 个关键步骤/概念，并带有连接箭头或流程
- **文本**：标签最少，字体大且易读
- 日志：`[HH:MM:SS] GENERATED: Graphical abstract for paper summary`

## 核心能力

### 1. 手稿结构和组织

**IMRAD 格式**：引导论文通过大多数科学学科使用的标准引言、方法、结果和讨论结构。这包括：
- **引言**：建立研究背景，识别空白，陈述目标
- **方法**：详细描述研究设计、参与者、程序和分析方法
- **结果**：客观呈现发现，不进行解释
- **讨论**：解释结果，承认局限性，提出未来方向

有关 IMRAD 结构的详细指导，请参考 `references/imrad_structure.md`。

**替代结构**：支持特定学科的格式，包括：
- 综述文章（叙述性、系统性、范围性）
- 病例报告和病例系列
- Meta 分析和汇总分析
- 理论/建模论文
- 方法论文和协议

### 2. 特定章节的写作指导

**摘要撰写**：撰写简洁、独立的摘要（100-250 字），捕捉论文的目的、方法、结果和结论。支持结构化摘要（带标签的部分）和非结构化单段格式。

**引言发展**：构建引人注目的引言：
- 确立研究问题的重要性
- 系统地回顾相关文献
- 识别知识空白或争议
- 陈述明确的研究问题或假设
- 解释研究的新颖性和重要性

**方法文档**：确保可重复性：
- 详细描述参与者/样本
- 清晰的程序文档
- 统计方法的正当性
- 设备和材料规格
- 伦理批准和知情同意声明

**结果呈现**：以以下方式呈现发现：
- 从主要结果到次要结果的逻辑流
- 与图表的整合
- 统计显著性和效应大小
- 客观报告，不进行解释

**讨论构建**：通过以下方式综合发现：
- 将结果与研究问题联系起来
- 与现有文献进行比较
- 诚实地承认局限性
- 提出机制解释
- 建议实际意义和未来研究

### 3. 引用和参考文献管理

在各学科中正确应用引用样式。有关全面的样式指南，请参考 `references/citation_styles.md`。

**主要引用样式：**
- **AMA（美国医学协会）**：带编号的上标引用，医学领域常用
- **Vancouver**：方括号中的编号引用，生物医学标准
- **APA（美国心理学协会）**：作者-日期的文本引用，社会科学常用
- **Chicago**：注释-书目或作者-日期，人文学科和科学
- **IEEE**：带编号的方括号，工程和计算机科学

**最佳实践：**
- 尽可能引用原始来源
- 包括最近的文献（活跃领域的最后 5-10 年）
- 在引言和讨论中平衡引用分布
- 核实所有引用与原始来源一致
- 使用参考管理软件（Zotero、Mendeley、EndNote）

### 4. 图形和表格

创建有效的数据可视化，以增强理解。有关详细的最佳实践，请参考 `references/figures_tables.md`。

**何时使用表格与图形：**
- **表格**：精确的数值数据、复杂数据集、需要确切值的多个变量
- **图形**：趋势、模式、关系、比较，视觉上更易理解

**设计原则：**
- 使每个表格/图形自我解释，附上完整的说明
- 在所有显示项目中使用一致的格式和术语
- 标记所有轴、列和行，并附上单位
- 包括样本量（n）和统计注释
- 遵循“每 1000 字一个表格/图形”的指导原则
- 避免在文本、表格和图形之间重复信息

**常见图形类型：**
- 条形图：比较离散类别
- 折线图：显示随时间变化的趋势
- 散点图：显示相关性
- 箱线图：显示分布和异常值
- 热图：可视化矩阵和模式

### 5. 按研究类型的报告指南

通过遵循既定的报告标准，确保完整性和透明度。有关全面的指南细节，请参考 `references/reporting_guidelines.md`。

**关键指南：**
- **CONSORT**：随机对照试验
- **STROBE**：观察性研究（队列、病例对照、横断面）
- **PRISMA**：系统评价和 Meta 分析
- **STARD**：诊断准确性研究
- **TRIPOD**：预测模型研究
- **ARRIVE**：动物研究
- **CARE**：病例报告
- **SQUIRE**：质量改进研究
- **SPIRIT**：临床试验的研究方案
- **CHEERS**：经济评估

每个指南提供检查清单，确保报告所有关键方法要素。

### 6. 写作原则和风格

应用基本的科学写作原则。有关详细指导，请参考 `references/writing_principles.md`。

**清晰性**：
- 使用精确、明确的语言
- 在首次使用时定义技术术语和缩写
- 保持段落内部和之间的逻辑流
- 在适当时使用主动语态以提高清晰度

**简洁性**：
- 消除冗余的词语和短语
- 偏好较短的句子（平均 15-20 个词）
- 删除不必要的限定词
- 严格遵守字数限制

**准确性**：
- 报告精确值，保持适当的精度
- 在整个文档中使用一致的术语
- 区分观察和解释
- 适当地承认不确定性

**客观性**：
- 无偏见地呈现结果
- 避免夸大发现或影响
- 承认相互矛盾的证据
- 保持专业、中立的语气

### 7. 写作过程：从大纲到完整段落

**关键：始终以完整段落写作，绝不要在科学论文中提交项目符号。**

科学论文必须以完整、流畅的散文写成。使用以下两阶段方法进行有效写作：

**阶段 1：创建包含关键点的章节大纲**

在开始新章节时：
1. 使用 research-lookup 技能收集相关文献和数据
2. 创建一个结构化的大纲，使用项目符号标记：
   - 要呈现的主要论点或发现
   - 需要引用的关键研究
   - 要包含的数据点和统计信息
   - 逻辑流和组织
3. 这些项目符号作为支架——它们不是最终手稿

**示例大纲（引言部分）：**
```
- 背景：AI 在药物发现中日益受到关注
  * 引用最近的综述（Smith 2023，Jones 2024）
  * 传统方法速度慢且成本高
- 空白：对罕见疾病的应用有限
  * 仅有 2 项先前研究（Lee 2022，Chen 2023）
  * 小数据集仍然是一个挑战
- 我们的方法：从常见疾病中迁移学习
  * 新颖的架构结合 X 和 Y
- 研究目标：在 3 个罕见疾病数据集上进行验证
```

**阶段 2：将关键点转换为完整段落**

一旦大纲完成，将每个项目符号扩展为适当的散文：

1. **将项目符号转换为完整句子**，包括主语、动词和宾语
2. **添加过渡**，使句子和思想之间流畅（然而、此外、相反、随后）
3. **自然地整合引用**，而不是作为列表
4. **扩展上下文和解释**，补充项目符号所遗漏的内容
5. **确保逻辑流**，从一个句子到下一个句子
6. **变化句子结构**，以保持读者的参与

**示例转换为散文：**

```
在过去十年中，人工智能方法在药物发现管道中获得了显著关注（Smith，2023；Jones，2024）。虽然这些计算方法显示出加速治疗候选药物识别的潜力，但传统实验方法仍然缓慢且资源密集，通常需要数年的实验室工作和大量的财务投资。然而，AI 在罕见疾病中的应用仍然有限，仅有两项先前研究展示了概念验证结果（Lee，2022；Chen，2023）。主要障碍是针对小患者群体的训练数据稀缺。

为了解决这一挑战，我们开发了一种迁移学习方法，利用来自良好表征的常见疾病的知识来预测罕见病的治疗靶点。我们新颖的神经架构结合了用于分子特征提取的卷积层和用于蛋白质-配体相互作用建模的注意机制。本研究的目标是在三个独立的罕见疾病数据集上验证我们的方法，评估预测准确性和结果的生物学可解释性。
```

**大纲与最终文本之间的关键区别：**

| 大纲（规划阶段） | 最终手稿 |
|------------------|----------|
| 项目符号和片段   | 完整的句子和段落 |
| 电报式笔记       | 具有上下文的完整解释 |
| 引用列表         | 引用自然地融入散文中 |
| 概念的简略描述   | 发展论点，带有过渡 |
| 仅供您参考      | 用于出版和同行评审 |

**避免的常见错误：**

- ❌ **绝不要** 在最终手稿中留下项目符号
- ❌ **绝不要** 提交应为段落的列表
- ❌ **不要** 在结果或讨论部分使用编号或项目符号列表（除非在特定情况下，如研究假设或纳入标准）
- ❌ **不要** 写句子片段或不完整的思想
- ✅ **请** 在方法部分偶尔使用列表（例如，纳入/排除标准、材料列表）
- ✅ **请** 确保每个部分都流畅地连接成散文
- ✅ **请** 大声朗读段落，以检查自然流畅性

**列表可接受的情况（有限案例）：**

列表仅可在科学论文中特定上下文中出现：
- **方法**：纳入/排除标准、材料和试剂、参与者特征
- **补充材料**：扩展协议、设备列表、详细参数
- **绝不要在**：摘要、引言、结果、讨论、结论中使用

**与研究查找的整合：**

research-lookup 技能对于阶段 1（创建大纲）至关重要：
1. 使用 research-lookup 搜索相关论文
2. 提取关键发现、方法和数据
3. 将发现组织为大纲中的项目符号
4. 然后在阶段 2 中将大纲转换为完整段落

这种两阶段过程确保您：
- 系统地收集和组织信息
- 在写作之前创建逻辑结构
- 生成抛光的、适合出版的散文
- 关注叙述流畅性

### 8. 期刊特定格式

根据期刊要求调整手稿：
- 遵循作者指南的结构、长度和格式
- 应用期刊特定的引用样式
- 满足图表的规格（分辨率、文件格式、尺寸）
- 包括所需的声明（资金、利益冲突、数据可用性、伦理批准）
- 遵守每个部分的字数限制
- 按照提供的模板要求进行格式化

### 9. 领域特定语言和术语

根据特定科学学科调整语言、术语和惯例。每个领域都有既定的词汇、首选的表达方式和领域特定的惯例，这些都表明专业性并确保目标受众的清晰理解。

**识别领域特定的语言惯例：**
- 回顾目标期刊中最近高影响力论文使用的术语
- 注意领域特定的缩写、单位和符号系统
- 确定首选术语（例如，“参与者”与“受试者”，“化合物”与“药物”，“标本”与“样本”）
- 观察方法、组织或技术的典型描述方式

**生物医学和临床科学：**
- 使用精确的解剖和临床术语（例如，在正式写作中使用“心肌梗死”而不是“心脏病发作”）
- 遵循标准化的疾病命名法（ICD、DSM、SNOMED-CT）
- 在首次提及药物名称时使用通用名称，必要时在括号中使用品牌名称
- 对于临床研究使用“患者”，对于社区基础研究使用“参与者”
- 遵循人类基因变异协会（HGVS）命名法报告基因变异
- 使用标准单位（SI 单位在大多数国际期刊中）报告实验室值

**分子生物学和遗传学：**
- 基因符号使用斜体（例如，*TP53*），蛋白质使用常规字体（例如，p53）
- 遵循特定物种的基因命名法（人类使用大写：*BRCA1*；小鼠使用句子大小写：*Brca1*）
- 在首次提及时完整指定生物体名称，然后使用公认的缩写（例如，*Escherichia coli*，然后使用 *E. coli*）
- 使用标准遗传符号（例如，+/+，+/-，-/- 表示基因型）
- 使用公认的术语描述分子技术（例如，“定量 PCR”或“qPCR”，而不是“实时 PCR”）

**化学和药物科学：**
- 遵循 IUPAC 化学化合物命名法
- 对于新化合物使用系统名称，对于知名物质使用常用名称
- 使用标准符号（例如，SMILES、InChI 用于数据库）报告化学结构
- 使用适当单位（mM、μM、nM 或 % w/v、v/v）报告浓度
- 使用公认的反应命名法描述合成路线
- 使用领域定义的一致术语（例如，“生物利用度”，“药代动力学”，“IC50”）

**生态学和环境科学：**
- 使用物种的双名法（斜体：*Homo sapiens*）
- 在相关时首次提及物种时指定分类权威
- 使用标准化的栖息地和生态系统分类
- 使用一致的术语描述生态指标（例如，“物种丰富度”，“香农多样性指数”）
- 使用公认的术语描述采样方法（例如，“样带”，“样方”，“标记-再捕获”）

**物理学和工程学：**
- 一致地遵循 SI 单位，除非领域惯例另有规定
- 使用物理量的标准符号（标量与向量、张量）
- 使用公认的术语描述现象（例如，“量子纠缠”，“层流”）
- 在相关时指定设备的型号和制造商
- 使用与领域标准一致的数学符号

**神经科学：**
- 使用标准化的大脑区域命名法（例如，参考 Allen Brain Atlas）
- 使用公认的立体定位系统指定大脑区域的坐标
- 遵循神经术语的惯例（例如，在正式写作中使用“动作电位”而不是“尖峰”）
- 使用“神经活动”、“神经元放电”、“大脑激活”适当地基于测量方法
- 使用适当的特异性描述记录技术（例如，“全细胞贴片钳”，“细胞外记录”）

**社会和行为科学：**
- 在适当时使用以人为本的语言（例如，“有精神分裂症的人”而不是“精神分裂症患者”）
- 使用标准化的心理学构念和经过验证的评估名称
- 遵循 APA 指南以减少语言偏见
- 使用公认的理论框架，采用标准术语
- 使用“参与者”而不是“受试者”进行人类研究

**一般原则：**

**匹配受众专业性：**
- 对于专业期刊：自由使用领域特定术语，仅在首次使用时定义高度专业或新颖的术语
- 对于广泛影响的期刊（例如 *Nature*、*Science*）：定义更多技术术语，为专业概念提供上下文
- 对于跨学科受众：平衡精确性与可接近性，在首次使用时定义术语

**战略性定义技术术语：**
- 在首次使用时定义缩写：“信使 RNA (mRNA)”
- 在为更广泛的受众写作时提供对专业技术的简要解释
- 避免在目标受众熟悉的情况下过度定义术语（信号表明对领域的不熟悉）
- 如果不可避免地使用大量专业术语，请创建词汇表

**保持一致性：**
- 在整个文档中对同一概念使用相同的术语（不要在“药物”、“药物”和“制药”之间交替）
- 在定义后决定使用“PCR”或“聚合酶链反应”的相同系统
- 在整个文档中遵循相同的命名法系统（尤其是对于基因、物种、化学物质）

**避免领域混合错误：**
- 不要对基础科学使用临床术语（例如，不要称小鼠为“患者”）
- 避免在精确领域术语的地方使用口语或过于一般的术语
- 不要在不确保适当使用的情况下引入相邻领域的术语

**验证术语使用：**
- 查阅领域特定的样式指南和命名资源
- 检查目标期刊中术语的使用情况
- 使用领域特定的数据库和本体（例如，基因本体、MeSH 术语）
- 当不确定时，引用建立术语的关键参考

### 10. 常见陷阱

**主要拒稿原因：**
1. 不适当、不完整或描述不足的统计数据
2. 结果的过度解释或不支持的结论
3. 描述不清的影响可重复性的方法
4. 样本小、偏倚或不适当
5. 写作质量差或难以理解的文本
6. 文献回顾或背景不足
7. 图表不清晰或设计不当
8. 未遵循报告指南

**写作质量问题：**
- 不当混合时态（方法/结果使用过去时，已建立事实使用现在时）
- 过多的行话或未定义的缩写
- 段落分隔破坏逻辑流
- 缺少部分之间的过渡
- 符号或术语不一致

## 手稿开发工作流程

**阶段 1：规划**
1. 确定目标期刊并审查作者指南
2. 确定适用的报告指南（CONSORT、STROBE 等）
3. 大纲手稿结构（通常为 IMRAD）
4. 计划图表作为论文的骨架

**阶段 2：草拟**（对每个部分使用两阶段写作过程）
1. 从图表和表格开始（核心数据故事）
2. 对于每个部分，遵循两阶段过程：
   - **第一**：使用 research-lookup 创建包含关键点的大纲
   - **第二**：将项目符号转换为完整段落，形成流畅的散文
3. 撰写方法（通常最容易草拟）
4. 草拟结果（客观描述图表/表格）
5. 撰写讨论（解释发现）
6. 撰写引言（设定研究问题）
7. 撰写摘要（综合完整故事）
8. 创建标题（简洁且描述性）

**请记住**：项目符号仅用于规划——最终手稿必须为完整段落。

**阶段 3：修订**
1. 检查整个文档的逻辑流和“红线”
2. 验证术语和符号的一致性
3. 确保图表/表格自我解释
4. 确认遵循报告指南
5. 核实所有引用的准确性和格式
6. 检查每个部分的字数
7. 校对语法、拼写和清晰度

**阶段 4：最终准备**
1. 根据期刊要求进行格式化
2. 准备补充材料
3. 撰写强调重要性的覆盖信
4. 完成提交检查清单
5. 收集所有所需声明和表格
