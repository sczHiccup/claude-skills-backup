# Scientific Slides

## 概述

科学演示是传达研究、分享发现和与学术及专业观众互动的重要媒介。本技能提供了创建有效科学演示的全面指导，从结构和内容开发到视觉设计和交付准备。

**关键重点**：会议、研讨会、答辩和专业演讲的口头演示。

**关键设计理念**：科学演示应具有视觉吸引力并以研究为基础。无论如何都要避免干巴巴、文字过多的幻灯片。优秀的科学演示结合了：
- **引人注目的视觉效果**：高质量的图形、图像、图表（不仅仅是要点）
- **研究背景**：适当的引用来自研究查阅以建立可信度
- **最小文本**：要点作为提示，您口头提供解释
- **专业设计**：现代配色方案、强大的视觉层次、充足的留白
- **以故事为驱动**：清晰的叙事弧，而不仅仅是数据堆砌

**记住**：无聊的演示 = 被遗忘的科学。在保持科学严谨性的同时，通过适当的引用使您的幻灯片在视觉上令人难忘。

## 何时使用此技能

当您需要时，应使用此技能：
- 准备会议演示（5-20分钟）
- 开发学术研讨会（45-60分钟）
- 创建论文或学位答辩演示
- 设计资助提案演示
- 准备期刊俱乐部演示
- 在机构或公司进行研究演讲
- 进行科学主题的教学或辅导演示

## 使用 Nano Banana Pro 生成幻灯片

**此技能使用 Nano Banana Pro AI 自动生成令人惊叹的演示幻灯片。**

根据输出格式，有两种工作流程：

### 默认工作流程：PDF 幻灯片（推荐）

使用 Nano Banana Pro 将每张幻灯片生成完整的图像，然后合并为 PDF。这将产生最具视觉冲击力的结果。

**工作原理：**
1. **规划幻灯片**：为每张幻灯片创建详细计划（标题、要点、视觉元素）
2. **生成幻灯片**：调用 Nano Banana Pro 为每张幻灯片创建完整的幻灯片图像
3. **合并为 PDF**：将幻灯片图像组装成一个 PDF 演示文稿

**步骤 1：规划每张幻灯片**

在生成之前，为您的演示创建详细计划：

```markdown
# 演示计划：机器学习简介

## 幻灯片 1：标题幻灯片
- 标题：“机器学习：从理论到实践”
- 副标题：“AI 会议 2025”
- 演讲者：Dr. Jane Smith，XYZ 大学
- 视觉：现代抽象神经网络背景

## 幻灯片 2：介绍
- 标题：“为什么机器学习重要”
- 要点：行业采用、突破性应用、未来潜力
- 视觉：显示不同机器学习应用的图标（医疗、金融、机器人）

## 幻灯片 3：核心概念
- 标题：“三种学习类型”
- 内容：监督学习、无监督学习、强化学习
- 视觉：三部分图表显示每种类型及示例

...（继续为所有幻灯片）
```

**步骤 2：生成每张幻灯片**

使用 `generate_slide_image.py` 脚本创建每张幻灯片。

**关键：格式一致性协议**

为了确保演示中所有幻灯片的格式统一：

1. **在演示开始时定义格式目标**，并在每个提示中包含：
   - 配色方案（例如，“深蓝色背景，白色文本，金色点缀”）
   - 排版风格（例如，“粗体无衬线标题，简洁的正文文本”）
   - 视觉风格（例如，“简约、专业、企业美学”）
   - 布局方式（例如，“充足的留白，左对齐内容”）

2. **在生成后续幻灯片时始终附加前一张幻灯片**，使用 `--attach`：
   - 这使得 Nano Banana Pro 能够看到并匹配现有样式
   - 在整个幻灯片中创建视觉连续性
   - 确保颜色、字体和设计语言的一致性

3. **默认作者为“K-Dense”**，除非指定其他名称

4. **对于引用研究的幻灯片，直接在提示中包含引用**：
   - 在提示文本中添加引用，以便它们出现在生成的幻灯片上
   - 使用格式：“包括引用：（作者等，年份）”或“显示参考：作者等，年份”
   - 对于多个引用，在提示中列出所有引用
   - 引用应以小文本出现在幻灯片底部或相关内容附近

5. **附加现有图形/数据以用于结果幻灯片**（对于数据驱动的演示至关重要）：
   - 在创建关于结果的幻灯片时，始终检查工作目录中是否存在现有图形：
     - 工作目录（例如，`figures/`、`results/`、`plots/`、`images/`）
     - 用户提供的输入文件或目录
     - 与演示相关的任何数据可视化、图表或图形
   - 使用 `--attach` 附加这些图形，以便 Nano Banana Pro 可以将其纳入：
     - 附加实际数据图形/图表以用于结果幻灯片
     - 附加相关图表以用于方法幻灯片
     - 附加标题幻灯片的徽标或机构图像
   - 在附加数据图形时，在提示中描述您希望的内容：
     - “创建一张幻灯片，展示附加的结果图表，并突出关键发现”
     - “围绕这张附加的图形构建一张幻灯片，添加标题和要点解释数据”
     - “将附加的图表纳入结果幻灯片并进行解释”
   - **在生成结果幻灯片之前**：列出工作目录中的文件以查找相关图形
   - 可以附加多个图形：`--attach fig1.png --attach fig2.png`

**带有格式一致性、引用和图形附加的示例：**

```bash
# 标题幻灯片（第一张幻灯片 - 确立样式）
python scripts/generate_slide_image.py "标题幻灯片：'机器学习：从理论到实践'。副标题：'AI 会议 2025'。演讲者：K-Dense。格式目标：深蓝色背景（#1a237e），白色文本，金色点缀（#ffc107），简约设计，无衬线字体，宽松边距，无装饰元素。" -o slides/01_title.png

# 内容幻灯片带引用（附加前一张幻灯片以保持一致性）
python scripts/generate_slide_image.py "演示幻灯片标题为'为什么机器学习重要'。三个要点，带简单图标：1）行业采用，2）突破性应用，3）未来潜力。引用：在底部小文本中包括： (LeCun et al., 2015; Goodfellow et al., 2016)。格式目标：与附加幻灯片样式完全匹配 - 深蓝色背景，白色文本，金色点缀，简约专业设计，无视觉杂乱。" -o slides/02_intro.png --attach slides/01_title.png

# 背景幻灯片带多个引用
python scripts/generate_slide_image.py "演示幻灯片标题为'深度学习革命'。关键里程碑：ImageNet 突破（2012），变压器架构（2017），GPT 模型（2018-现在）。引用：底部显示参考： (Krizhevsky et al., 2012; Vaswani et al., 2017; Brown et al., 2020)。格式目标：与附加幻灯片样式完全匹配 - 相同颜色，字体，简约设计。" -o slides/03_background.png --attach slides/02_intro.png

# 结果幻灯片 - 附加工作目录中的实际数据图形
# 首先，检查存在的图形：ls figures/ 或 ls results/
python scripts/generate_slide_image.py "演示幻灯片标题为'模型性能结果'。创建一张幻灯片，展示附加的准确性图表。关键发现突出：1）达到 95% 准确率，2）比基线高出 12%，3）在测试集上保持一致。引用：底部包括： (我们的结果，2025)。格式目标：与附加幻灯片样式完全匹配。" -o slides/04_results.png --attach slides/03_background.png --attach figures/accuracy_chart.png

# 结果幻灯片 - 多个图形比较
python scripts/generate_slide_image.py "演示幻灯片标题为'前后比较'。使用两个附加图形构建并排比较幻灯片。左侧：基线结果，右侧：我们的改进结果。添加简要标签解释改进。格式目标：与附加幻灯片样式完全匹配。" -o slides/05_comparison.png --attach slides/04_results.png --attach figures/baseline.png --attach figures/improved.png

# 方法论幻灯片 - 附加现有图表
python scripts/generate_slide_image.py "演示幻灯片标题为'系统架构'。展示附加的架构图，并附上简要说明要点：1）输入处理，2）模型推理，3）输出生成。格式目标：与附加幻灯片样式完全匹配。" -o slides/06_architecture.png --attach slides/05_comparison.png --attach diagrams/system_architecture.png
```

**重要提示：在创建结果幻灯片之前，始终：**
1. 列出工作目录中的文件：`ls -la figures/` 或 `ls -la results/`
2. 检查用户提供的目录以查找相关图形
3. 附加所有应出现在幻灯片上的相关图形
4. 描述 Nano Banana Pro 应如何纳入附加图形

**提示模板：**

在每个提示中包含这些元素（根据需要自定义）：
```
[幻灯片内容描述]
引用：底部包括：（作者1 et al., 年份；作者2 et al., 年份）
格式目标：[背景颜色]，[文本颜色]，[点缀颜色]，简约专业设计，无装饰元素，与附加幻灯片样式一致。
```

使用 Nano Banana Pro AI 生成演示幻灯片或视觉效果。

# 完整幻灯片（默认） - 生成完整幻灯片作为图像

- `--api-key`：OpenRouter API 密钥（或设置 OPENROUTER_API_KEY 环境变量）
- `-v, --verbose`：详细输出

**附加参考图像：**

- “创建一张关于这些数据的幻灯片” + 附加数据图表
- “制作一张带有此徽标的标题幻灯片” + 附加徽标
- “将这些图形合并为一张幻灯片” + 附加多个图像
- “在幻灯片中解释此图表” + 附加图表

**环境设置：**

## 核心能力

### 1. 演示结构和组织

构建具有清晰叙事流和适当结构的演示，以适应不同的上下文。有关详细指导，请参阅 `references/presentation_structure.md`。

**通用故事弧**：
1. **吸引注意**：抓住注意力（30-60秒）
2. **背景**：建立重要性（演讲的 5-10%）
3. **问题/差距**：识别未知（演讲的 5-10%）
4. **方法**：解释您的解决方案（演讲的 15-25%）
5. **结果**：呈现关键发现（演讲的 40-50%）
6. **影响**：讨论意义（演讲的 15-20%）
7. **结尾**：令人难忘的结论（1-2分钟）

**特定演讲结构**：
- **会议演讲（15分钟）**：专注于 1-2 个关键发现，方法最少
- **学术研讨会（45分钟）**：全面覆盖，详细方法，多项研究
- **论文答辩（60分钟）**：完整的学位论文概述，涵盖所有研究
- **资助提案（15分钟）**：强调重要性、可行性和影响
- **期刊俱乐部（30分钟）**：对已发表工作的批判性分析

### 2. 幻灯片设计原则

创建专业、可读和可访问的幻灯片，以增强理解。有关完整设计指南，请参阅 `references/slide_design_principles.md`。

**反模式：避免干燥、文字过多的演示**

❌ **使演示干燥和令人遗忘的原因：**
- 文字墙（每张幻灯片超过 6 个要点）
- 小字体（<24pt 正文文本）
- 仅黑色文本在白色背景上（没有视觉兴趣）
- 没有图像或图形（仅要点）
- 没有自定义的通用模板
- 密集的段落式要点
- 缺少研究背景（没有引用）
- 所有幻灯片看起来相同（重复）

✅ **使演示引人入胜和令人难忘的原因：**
- 高质量视觉效果主导（图形、照片、图表、图标）
- 大而清晰的文本作为强调（而不是主要内容）
- 现代、目的明确的配色方案（而不是默认主题）
- 充足的留白（幻灯片呼吸）
- 以研究为基础的背景（来自研究查阅的适当引用）
- 幻灯片布局多样化（不是所有的要点列表）
- 以故事为驱动的流动，带有视觉锚
- 专业、精致的外观

**核心设计原则**：

**视觉优先方法**（至关重要）：
- 从视觉开始（图形、图像、图表），添加文本作为支持
- 每张幻灯片都应有强大的视觉元素（图形、图表、照片、图表）
- 文本解释或补充视觉，而不是替代它
- 思考：“我如何能展示这个，而不仅仅是告诉它？”
- 目标：60-70% 视觉内容，30-40% 文本

**简约而有影响力**：
- 每张幻灯片一个主要思想
- 最小文本（每个要点 3-4 个，最好 4-6 个单词）
- 充足的留白（幻灯片的 40-50%）
- 清晰的视觉焦点
- 大胆、自信的设计选择

**排版以吸引注意**：
- 无衬线字体（Arial、Calibri、Helvetica）
- 大字体：正文文本 24-28pt（而不是最低 18pt）
- 幻灯片标题 36-44pt（加粗）
- 高对比度（最低 4.5:1，最好 7:1）
- 使用大小来建立层次，而不仅仅是粗细

**颜色以产生影响**：
- 现代配色方案（而不是默认的蓝色/灰色）
- 考虑您的主题：生物技术？鲜艳的颜色。物理学？流畅的深色。健康？温暖的色调。
- 限制调色板（总共 3-5 种颜色）
- 高对比度组合
- 色盲安全（避免红绿色组合）
- 有目的地使用颜色（而不是装饰）

**布局以增加视觉兴趣**：
- 变化布局（不是所有的要点列表）
- 使用两列布局（文本 + 图形）
- 关键结果的全幻灯片图形
- 不对称构图（比居中更有趣）
- 三分之一法则用于焦点
- 一致但不重复

### 3. 幻灯片的数据可视化

根据演示上下文调整科学图形。有关详细指导，请参阅 `references/data_visualization_slides.md`。

**与期刊图形的关键区别**：
- 简化，而不是复制
- 更大的字体（18-24pt 最低）
- 更少的面板（分散在幻灯片上）
- 直接标记（而不是图例）
- 通过颜色和大小强调
- 复杂数据的逐步披露

**可视化最佳实践**：
- **条形图**：比较离散类别
- **折线图**：趋势和轨迹
- **散点图**：关系和相关性
- **热图**：矩阵数据和模式
- **网络图**：关系和连接

**常见错误需避免**：
- 字体太小（<18pt）
- 一张幻灯片上有太多面板
- 复杂的图例
- 对比度不足
- 布局杂乱

### 6. 视觉审查和迭代

通过视觉检查实施迭代改进。有关完整工作流程，请参阅 `references/visual_review_workflow.md`。

**视觉验证工作流程**：

**步骤 1：生成 PDF**（如果尚未生成 PDF）
- PowerPoint：导出为 PDF
- Beamer：编译 LaTeX 源

**步骤 2：转换为图像**
```bash
# 使用 pdf_to_images 脚本
python scripts/pdf_to_images.py presentation.pdf review/slide --dpi 150

# 或使用 pptx 技能的缩略图工具
python ../document-skills/pptx/scripts/thumbnail.py presentation.pptx review/thumb
```

**步骤 3：系统检查**

检查每张幻灯片：
- **文本溢出**：文本在边缘被截断
- **元素重叠**：文本与图像或其他文本重叠
- **字体大小**：文本太小（<18pt）
- **对比度**：文本与背景之间的对比度不足
- **布局问题**：对齐不当，间距差
- **视觉质量**：图像像素化，渲染不良

**步骤 4：记录问题**

创建问题日志：
```
幻灯片 # | 问题类型 | 描述 | 优先级
----------|----------|------|--------
3         | 文本溢出 | 要点 4 超出框 | 高
7         | 重叠     | 图形与标题重叠 | 高
12        | 字体大小 | 坐标标签太小 | 中
```

**步骤 5：应用修复**

对源文件进行更正：
- PowerPoint：编辑文本框，调整元素大小
- Beamer：调整 LaTeX 代码，重新编译

**步骤 6：重新验证**

重复步骤 1-5，直到没有关键问题。

**停止标准**：
- 没有文本溢出
- 没有不当重叠
- 所有文本可读（≥18pt）
- 充足的对比度（≥4.5:1）
- 专业外观

### 7. 时间和节奏

确保演示适合分配的时间。有关全面的时间指导，请参阅 `assets/timing_guidelines.md`。

- 一般指导：~1 张幻灯片每分钟
- 对于复杂幻灯片（2-3分钟）
- 对于简单幻灯片（15-30秒）

**时间分配**：
- 介绍：15-20%
- 方法：15-20%
- 结果：40-50%（最多时间）
- 讨论：15-20%
- 结论：5%

- 5分钟演讲：练习 5-7 次
- 15分钟演讲：练习 3-5 次
- 45分钟演讲：练习 3-4 次
- 答辩：练习 4-6 次

**时间检查点**：

对于 15 分钟的演讲：
- 3-4 分钟：完成介绍
- 7-8 分钟：结果过半
- 12-13 分钟：开始结论

**应急策略**：
- 跑得太快：跳过备用幻灯片（提前准备）
- 跑得太慢：扩展示例，稍微放慢
- 永远不要跳过结论

## 演示开发工作流程

### 阶段 1：规划（创建幻灯片之前）

**定义上下文**：
1. 演讲类型是什么？（会议、研讨会、答辩等）
2. 多长时间？（分钟数）
3. 观众是谁？（专业人士、普通观众、混合）
4. 场地是什么？（房间大小、音视频设置、虚拟/现场）
5. 之后会发生什么？（问答、讨论、网络交流）

**研究和文献回顾**（使用研究查阅技能）：
1. **搜索背景文献**：找到 5-10 篇关键论文以建立背景
2. **识别知识差距**：使用研究查阅找出未知的内容
3. **定位比较研究**：找到具有相似方法或结果的论文
4. **收集支持引用**：收集支持您解释的论文
5. **建立参考列表**：为幻灯片创建 .bib 文件或引用列表
6. **记录关键发现以引用**：记录特定结果以供参考

**开发内容大纲**：
1. 确定 1-3 个核心信息
2. 选择要展示的关键发现
3. 选择必要的图形（通常 3-6 个用于 15 分钟的演讲）
4. 规划叙事弧并适当引用
5. 按部分分配时间

**15分钟演讲的大纲示例**：
```
1. 标题（30秒）
2. 吸引点：引人注目的问题（60秒）[引用 1-2 篇论文通过研究查阅]
3. 背景（90秒）[引用 3-4 篇关键论文建立背景]
4. 研究问题（45秒）[引用显示差距的论文]
5. 方法概述（2分钟）
6-8. 主要结果 1（3分钟，3张幻灯片）
9-10. 主要结果 2（2分钟，2张幻灯片）
11-12. 结果 3 或验证（2分钟，2张幻灯片）
13-14. 讨论和影响（2分钟）[与 2-3 项先前研究进行比较]
15. 结论（45秒）
16. 致谢（15秒）

注意：在创建幻灯片之前，使用研究查阅查找背景（幻灯片 2-4）和讨论（幻灯片 13-14）的论文。
```

### 阶段 2：设计和创建

**选择实施方法**：

**选项 A：PowerPoint（通过 PPTX 技能）**
1. 阅读 `assets/powerpoint_design_guide.md`
2. 阅读 `document-skills/pptx/SKILL.md`
3. 选择方法（程序化或基于模板）
4. 创建具有一致设计的母版幻灯片
5. 根据大纲构建演示文稿

**选项 B：LaTeX Beamer**
1. 阅读 `references/beamer_guide.md`
2. 从 `assets/` 选择适当的模板
3. 自定义主题和颜色
4. 在 LaTeX 中编写内容
5. 编译为 PDF

**设计考虑**（使其在视觉上吸引人）：
- **选择现代配色方案**：与您的主题匹配（生物技术=鲜艳，物理学=流畅，健康=温暖）
  - 使用 pptx 技能的配色方案示例（青色与珊瑚、鲜艳红色、深紫色与翡翠等）
  - 不要仅仅使用默认的蓝色/灰色主题
  - 限制调色板（总共 3-5 种颜色）
- **选择干净的字体**：无衬线，较大尺寸（24pt+）
- **规划视觉元素**：每张幻灯片要包含哪些图像、图表、图标？
- **创建多样化的布局**：混合全图、两列、文本覆盖（而不是所有要点）
- **设计部分分隔符**：用引人注目的图形进行视觉分隔
- **规划动画/构建**：控制复杂幻灯片的信息流
- **增加视觉兴趣**：背景图像、颜色块、形状、图标

### 阶段 3：内容开发

**填充幻灯片**（视觉优先策略）：
1. **从视觉开始**：规划每个关键点的图形、图像、图表
2. **广泛使用研究查阅**：找到 8-15 篇论文以进行适当引用
3. **首先创建视觉骨架**：添加所有图形、图表、图像、图表
4. **添加最小文本作为支持**：要点补充视觉，而不是替代它
5. **设计部分分隔符**：用图像或图形进行视觉分隔（而不仅仅是文本）
6. **润色标题/结尾**：使其在视觉上引人注目，包含联系信息
7. **添加过渡/构建**：控制信息流

**视觉内容要求**（使幻灯片引人入胜）：
- **图像**：使用高质量的照片、插图、概念图形
- **图标**：概念的视觉表示（而不是装饰）
- **图表**：流程图、示意图、过程图
- **图形**：简化的研究图形，带有大标签（18-24pt 最低）
- **图表**：清晰的数据可视化，传达明确的信息
- **图形**：视觉隐喻、概念插图
- **颜色块**：使用彩色形状来视觉上组织内容
- 目标：每张幻灯片至少 1-2 个强大的视觉元素

**科学内容**（以研究为基础）：
- **引用**：广泛使用研究查阅找到相关论文
  - 介绍：引用 3-5 篇论文建立背景和差距
  - 背景：以视觉方式展示关键先前工作（而不仅仅是引用）
  - 讨论：引用 3-5 篇论文与您的结果进行比较
  - 使用作者-年份格式（Smith et al., 2023）以提高可读性
  - 引用建立可信度和科学严谨性
- **图形**：从论文简化，标签大（18-24pt 最低）
- **方程**：大而清晰，解释每个术语（尽量少用）
- **表格**：最小化，突出关键比较（而不是数据倾倒）
- **代码/算法**：使用语法高亮，保持简短

**文本指南**（少即是多）：
- 项目符号，绝不段落
- 每张幻灯片 3-4 个要点（最多 6 个仅在必要时）
- 每个要点 4-6 个单词（比 6×6 规则更短）
- 关键术语加粗
- 文本是支持角色，视觉是明星
- 使用构建来控制节奏

## 常见陷阱

### 内容错误

**干燥、无聊的演示**（至关重要避免）：
- 问题：文字过多的幻灯片没有视觉兴趣，缺少研究背景
- 征兆：所有要点，没有图像，没有图形，默认模板，没有引用
- 解决方案：
  - 使用研究查阅找到 8-15 篇论文以提供可信的背景
  - 在每张幻灯片上添加高质量的视觉效果（图形、照片、图表、图标）
  - 选择与主题相符的现代配色方案
  - 变化幻灯片布局（不是所有的要点列表）
  - 通过视觉讲述故事，文本要点要少

**内容过多**：
- 问题：试图包含论文中的所有内容
- 解决方案：对于短演讲，专注于 1-2 个关键发现，视觉展示

**文本过多**：
- 问题：幻灯片上有完整段落，密集的要点，逐字阅读
- 解决方案：每张幻灯片 3-4 个要点，每个要点 4-6 个单词，让视觉传达信息

**缺少研究背景**：
- 问题：没有引用，声明没有支持，定位不清
- 解决方案：使用研究查阅找到论文，在介绍中引用 3-5 篇，在讨论中引用 3-5 篇

**叙事不清**：
- 问题：主题之间跳跃，没有清晰的故事，没有流动性
- 解决方案：遵循故事弧，使用视觉过渡，保持线索

**结果部分匆忙**：
- 问题：简要的方法，简要的结果，冗长的讨论
- 解决方案：在结果上花费 40-50% 的时间，视觉展示数据

### 设计错误

**通用、默认外观**：
- 问题：使用默认 PowerPoint/Beamer 主题而没有自定义，看起来过时
- 解决方案：选择现代配色方案，自定义字体/布局，增加视觉个性

**文本密集、视觉稀缺**：
- 问题：所有要点幻灯片，没有图像或图形，乏味
- 解决方案：在每张幻灯片上添加图形、照片、图表、图标，使其在视觉上吸引人

**小字体**：
- 问题：正文文本 <18pt，后面不可读，看起来不专业
- 解决方案：24-28pt 的字体（而不是最低 18pt），标题 36-44pt（加粗）

**低对比度**：
- 问题：浅色文本在浅色背景上，能见度差，难以阅读
- 解决方案：高对比度（最好 7:1，最低 4.5:1），使用对比度检查器进行测试

**幻灯片杂乱**：
- 问题：元素太多，缺乏留白，令人不知所措
- 解决方案：每张幻灯片一个主要思想，40-50% 留白，宽松间距

**格式不一致**：
- 问题：幻灯片之间字体、颜色、布局不同，看起来不专业
- 解决方案：使用母版幻灯片，保持设计系统，确保专业一致性

**缺少视觉层次**：
- 问题：所有内容大小和颜色相同，没有强调，焦点不清
- 解决方案：大小差异（标题大，正文中等），颜色强调，清晰焦点

### 时间错误

**不练习**：
- 问题：第一次演练是在演示中
- 解决方案：至少练习 3 次，使用计时器

**没有时间检查点**：
- 问题：直到太晚才意识到超时
- 解决方案：设置 3-4 个检查点，随时监控

**超时**：
- 问题：极不专业，影响问答时间
- 解决方案：准确练习，准备计划 B（跳过幻灯片）

**跳过结论**：
- 问题：超时，匆忙结束或跳过结尾
- 解决方案：永远不要跳过结论，而是剪掉早期内容

## 快速入门指南

### 对于 15 分钟的会议演讲（PDF 工作流程 - 推荐）

1. **研究与规划**（45分钟）：
   - **使用研究查阅**找到 8-12 篇相关论文以进行引用
   - 建立参考列表（背景、比较研究）
   - 大纲内容（介绍 → 方法 → 2-3 个关键结果 → 结论）
   - **为每张幻灯片创建详细计划**（标题、要点、视觉元素）
   - 目标 15-18 张幻灯片

2. **使用 Nano Banana Pro 生成幻灯片**（1-2小时）：

   **重要提示：使用一致的格式，附加前一张幻灯片，并包含引用！**

   ```bash
   # 标题幻灯片（确立样式 - 默认作者：K-Dense）
   python scripts/generate_slide_image.py "标题幻灯片：'您的研究标题'。会议名称，K-Dense。格式目标：[您的配色方案]，简约专业设计，无装饰元素，干净和企业化。" -o slides/01_title.png
   
   # 介绍幻灯片带引用（附加前一张以保持一致性）
   python scripts/generate_slide_image.py "幻灯片标题为'为什么这很重要'。三个要点，带简单图标。引用：在底部小文本中包括：（Smith et al., 2023; Jones et al., 2024）。格式目标：与附加幻灯片样式完全匹配。" -o slides/02_intro.png --attach slides/01_title.png
   
   # 继续为每张幻灯片（始终附加前一张，引用相关内容）
   python scripts/generate_slide_image.py "幻灯片标题为'方法'。关键方法要点。引用：（基于 Chen et al., 2022）。格式目标：与附加幻灯片样式完全匹配。" -o slides/03_methods.png --attach slides/02_intro.png
   
   # 合并为 PDF
   python scripts/slides_to_pdf.py slides/*.png -o presentation.pdf
   ```

3. **审查与迭代**（30分钟）：
   - 打开 PDF 并审查每张幻灯片
   - 重新生成需要改进的幻灯片
   - 重新合并为 PDF

4. **练习**（2-3小时）：
   - 练习 3-5 次，使用计时器
   - 目标 13-14 分钟（留出缓冲时间）
   - 录制自己，观看回放
   - **准备问题**（使用研究查阅进行预判）

5. **最终确认**（30分钟）：
   - 如果需要，生成备份/附录幻灯片
   - 保存多个副本
   - 在演示计算机上测试

总时间：~5-6 小时，生成高质量的 AI 演示文稿

## 总结：关键原则

1. **视觉优先设计**：每张幻灯片需要强大的视觉元素（图形、图像、图表） - 避免仅文本幻灯片
2. **以研究为基础**：使用研究查阅找到 8-15 篇论文，在介绍中引用 3-5 篇，在讨论中引用 3-5 篇
3. **现代美学**：选择与主题相符的现代配色方案，而不是默认主题
4. **最小文本**：3-4 个要点，每个要点 4-6 个单词（24-28pt 字体），让视觉讲述故事
5. **结构**：遵循故事弧，在结果上花费 40-50% 的时间
6. **高对比度**：专业外观，最好 7:1
7. **多样化布局**：混合全图、两列、视觉覆盖（而不是所有要点）
8. **时间**：练习 3-5 次，~1 张幻灯片每分钟，永远不要跳过结论
9. **验证**：视觉审查工作流程以捕捉溢出和重叠
10. **留白**：幻灯片的 40-50% 空白，以便视觉呼吸

**记住**：
- **无聊 = 被遗忘**：干燥、文字过多的幻灯片无法传达您的科学
- **视觉 + 研究 = 影响**：结合引人注目的视觉效果和以研究为基础的背景
- **您是演示，幻灯片是视觉支持**：它们应增强，而不是替代您的演讲
